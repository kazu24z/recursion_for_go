## 惑星での体重 easy
Aaron は、宇宙探査のための惑星探査機を設計している宇宙工学者です。彼は、探査機が異なる惑星でどのように機能するかを理解するために、各惑星の重力が探査機に与える影響を調査する必要があります。惑星ごとに重力が異なるため、同じ質量の物体でも重さが変わります。この重さの違いを正確に理解し計算することは、探査機の設計において非常に重要です。

Aaronの最初のアプローチは、地球、火星、木星それぞれの惑星に対して個別の関数を作成することでした。これらの関数は、与えられた質量に基づいて、それぞれの惑星における物体の重さを計算し、結果をコンソールに出力します。しかし、この方法にはいくつかの問題点がありました。新しい惑星を追加する際、または惑星の重力が変更された際に、それぞれの関数を個別に更新する必要があり、これは時間の無駄であり効率的ではありませんでした。

このシナリオでは、ソフトウェア開発におけるモジュール性と効率の問題を示しています。Aaron は後に、高階関数を用いることで、より効率的で再利用性の高いコードを書くことを学びます。weightFormulaByPlanet という高階関数を使うことで、異なる惑星の重力に基づいて質量から重さを計算する関数を生成できるようになりました。この方法により、新しい惑星の追加や既存の惑星の重力の更新が簡単になり、コードの重複を減らし、将来的な変更や拡張が容易になります。Aaron のこの取り組みは、宇宙探査ミッションにおいて重要な役割を果たし、探査機の設計に大きく貢献しました。

質量を引数として受け取り、その惑星における重さを計算してメッセージとして出力する、以下のラムダ関数を作成することにしました。計算結果については、小数点以下は切り捨ててください。

void getWeightOnEarth(int weight)
質量を受け取ると、地球上での重さを出力するラムダ関数。

void getWeightOnMars(int weight)
質量を受け取ると、火星上での重さを出力するラムダ関数。

void getWeightOnJupiter(int weight)
質量を受け取ると、木星上での重さを出力するラムダ関数。

これらのラムダ関数は、それぞれの惑星に特化した計算を行うため、weightFormulaByPlanet(string planet) という関数を使用して生成されます。この高階関数は、指定された惑星名に基づいて適切な重量計算式を持つラムダ関数を返します。Aaron のこの取り組みは、探査機の設計において重要な役割を果たし、異なる惑星の環境での機能を理解するのに役立ちます。

以下の重力加速度を計算に使用してください。

|惑星名|重力加速度[m/s2]|  
|:--:|:--|
|Earth|9.8|  
|Mars|3.7|  
|Jupiter|24.8|  


関数の入出力例
```
getWeightOnEarth = weightFormulaByPlanet("Earth")

getWeightOnEarth(50) --> The weight on Earth is 490


getWeightOnMars = weightFormulaByPlanet("Mars")

getWeightOnMars(70) --> The weight on Mars is 259


getWeightOnJupiter = weightFormulaByPlanet("Jupiter")

getWeightOnJupiter(90) --> The weight on Jupiter is 2232

```
